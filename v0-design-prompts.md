# v0プロフェッショナルデザイン改善プロンプト

## 📋 プロジェクト概要

**Tomorrow イベントプラットフォーム**
イベント出店者と主催者をつなぐB2Bプラットフォーム。出店者の登録・審査、イベント掲載・応募を一元管理。

---

## 🎨 デザイン要件（全体）

### デザインの方向性
- **プロフェッショナル**: 企業・ビジネス向けの信頼感
- **モダン**: 2024年のWebデザイントレンド
- **クリーン**: 余白を活かした見やすいUI
- **直感的**: ユーザーが迷わない導線設計

### デザイン原則
1. **一貫性**: カラー、フォント、コンポーネントの統一
2. **階層**: 明確な情報の優先順位
3. **余白**: 適切なスペーシング（16px基準）
4. **視認性**: WCAG AA準拠のコントラスト比
5. **レスポンシブ**: モバイルファースト

## 📱 コンポーネント別プロンプト

### 1. WelcomeScreen（ログイン画面） - 全アプリ共通

```
プロフェッショナルなログイン・新規登録画面を作成してください。

【デザイン要件】
- 左側: ブランディングエリア（デスクトップ）
  - グラデーション背景
  - プロダクト名とキャッチコピー
  - イラストまたはアイコン
- 右側: 認証フォーム
  - カード形式（影付き、丸角）
  - タブ切り替え（ログイン/新規登録）
  - モダンなフォームデザイン

【機能】
- メールアドレス/パスワード入力
- ソーシャルログイン（Google、LINE）
- パスワード表示/非表示トグル
- 「パスワードを忘れた方」リンク
- エラーメッセージ（フォーム下部、赤背景）
- ローディング状態

【レイアウト】
- デスクトップ: 左右2カラム（50:50）
- タブレット: 中央寄せ、ブランディングは上部に簡略化
- モバイル: 1カラム、ブランディングは最小限

【デザインのポイント】
- グラデーション: primary → accent（対角線）
- ボタン: ホバー時に色が濃くなる + 軽い影
- フォーム: フォーカス時にborder色がprimaryに変化
- アイコン: lucide-reactを使用
- トランジション: all 0.2s ease

【技術スタック】
Next.js 14, TypeScript, Tailwind CSS, shadcn/ui

【参考イメージ】
- Linear（https://linear.app）のログイン画面
- Notion（https://notion.so）のサインアップ
- Vercel（https://vercel.com）のログイン

【カラーバリエーション】
- Store: Primary #5DABA8
- Organizer: Primary #E58A7B
- Admin: Primary #3B82F6
```

### 2. ダッシュボード（ホーム画面）

```
モダンなダッシュボードを作成してください。

【デザイン要件】
- サイドバーナビゲーション（デスクトップ）
  - ロゴ（上部）
  - メニュー項目（アイコン + テキスト）
  - ユーザープロフィール（下部）
  - 折りたたみ可能
- トップバー
  - ページタイトル
  - 検索バー（中央）
  - 通知アイコン、プロフィールメニュー（右）
- メインコンテンツエリア
  - カードベースのレイアウト
  - グリッドシステム（2-4カラム）
  - 適切な余白とシャドウ

【コンポーネント】
- 統計カード（KPI表示）
  - 数値（大きく）
  - ラベル（小さく）
  - 増減インジケーター（↑↓）
  - 微妙なグラデーション背景
- アクションカード
  - アイコン（左または上）
  - タイトルと説明
  - CTAボタン
- リストビュー（最近の活動など）
  - アバター/アイコン
  - タイトルとサブタイトル
  - タイムスタンプ
  - ステータスバッジ

【レイアウト】
- デスクトップ: サイドバー（240px） + メインコンテンツ
- タブレット: サイドバー（アイコンのみ、60px） + メインコンテンツ
- モバイル: ボトムナビゲーション + メインコンテンツ

【デザインのポイント】
- カード: bg-white, shadow-sm, rounded-lg, hover:shadow-md
- アイコンサイズ: 20px（UI）、24px（メイン）
- 余白: p-4（カード内）、gap-4（グリッド）
- トランジション: スムーズなホバー効果

【技術スタック】
Next.js 14, TypeScript, Tailwind CSS, shadcn/ui, lucide-react

【参考イメージ】
- Stripe Dashboard
- GitHub Dashboard
- Vercel Dashboard
```

### 3. イベントカード（一覧画面）

```
魅力的なイベントカードコンポーネントを作成してください。

【デザイン要件】
- カード形式
  - イベント画像（上部、アスペクト比16:9）
  - コンテンツエリア（下部）
  - ホバー時: 軽く浮き上がる + 影が濃くなる
- イメージエリア
  - グラデーションオーバーレイ（下部）
  - ステータスバッジ（右上）: 募集中/募集終了/開催終了
  - お気に入りボタン（右上）
- コンテンツエリア
  - イベント名（font-semibold, text-lg）
  - 開催日時（アイコン + テキスト）
  - 開催場所（アイコン + テキスト）
  - 募集人数（アイコン + テキスト）
  - タグ（ジャンル、最大3つ）
  - 主催者アバター + 名前（下部）
  - 詳細ボタン（右下）

【レイアウト】
- グリッド: 
  - デスクトップ: 3カラム
  - タブレット: 2カラム
  - モバイル: 1カラム
- カードサイズ: 固定高さ（400px）

【デザインのポイント】
- 画像: object-cover, overflow-hidden
- バッジ: rounded-full, px-3 py-1, text-xs font-medium
- タグ: border付き、primaryカラー、ホバー時塗りつぶし
- アイコン: 16px、text-secondaryカラー
- 影: shadow-sm → shadow-lg（ホバー時）

【状態管理】
- デフォルト: 標準表示
- ホバー: translate-y-[-4px], shadow-lg
- ローディング: スケルトンスクリーン
- エラー: プレースホルダー画像

【技術スタック】
Next.js 14, TypeScript, Tailwind CSS, shadcn/ui, lucide-react, next/image

【参考イメージ】
- Airbnb リスティングカード
- Eventbrite イベントカード
- Peatix イベント表示
```

### 4. フォーム（多段階フォーム）

```
モダンな多段階フォームを作成してください。

【デザイン要件】
- プログレスインジケーター（上部）
  - ステップ番号 + タイトル
  - 完了ステップは緑のチェックマーク
  - 現在のステップはprimaryカラーで強調
  - 未完了ステップはグレー
  - ステップ間は線で接続
- フォームコンテンツ
  - カード形式、中央寄せ
  - 最大幅: 600px
  - フォームフィールド
    - ラベル（font-medium, mb-1.5）
    - インプット（border, focus:ring-2）
    - ヘルプテキスト（text-sm, text-secondary）
    - エラーメッセージ（text-error, text-sm）
- ナビゲーションボタン
  - 「戻る」（左、二次ボタン）
  - 「次へ」または「送信」（右、プライマリボタン）
  - モバイル: フル幅スタック

【入力タイプ】
- テキスト入力: 標準的なinput
- セレクトボックス: カスタムドロップダウン
- ファイルアップロード: ドラッグ&ドロップエリア
- チェックボックス/ラジオ: カスタムスタイル
- 日付ピッカー: カレンダーUI

【バリデーション】
- リアルタイムバリデーション
- エラー時: inputのborderが赤に変化
- 成功時: 緑のチェックマークアイコン
- 必須項目: ラベルに赤いアスタリスク

【デザインのポイント】
- インプット: h-10, px-3, rounded-md
- フォーカス: ring-2, ring-primary, ring-offset-2
- 余白: フィールド間は mb-4
- アニメーション: ステップ切り替え時にスライド

【技術スタック】
Next.js 14, TypeScript, Tailwind CSS, shadcn/ui, React Hook Form, zod

【参考イメージ】
- Typeform
- Linear のオンボーディング
- Cal.com の予約フォーム
```

### 5. テーブル（データ一覧）

```
プロフェッショナルなテーブルコンポーネントを作成してください。

【デザイン要件】
- ヘッダー
  - 背景: bg-gray-50
  - テキスト: font-medium, text-gray-700
  - ソート可能カラムにアイコン（↑↓）
  - ホバー時: bg-gray-100
- 行
  - 偶数行: 軽い背景色（ゼブラストライプ）
  - ホバー時: bg-gray-50
  - 選択可能な場合: チェックボックス（左端）
  - アクションボタン（右端）
- セル
  - 適切なpadding（px-4 py-3）
  - テキストオーバーフロー対応
  - ステータスはバッジで表示

【機能】
- ソート（昇順/降順）
- フィルター（カラムごと）
- 検索（全文検索）
- ページネーション（下部）
- 行選択（一括操作用）
- アクションメニュー（編集/削除など）

【レスポンシブ】
- デスクトップ: 標準テーブル
- タブレット: 横スクロール
- モバイル: カード形式に変換

【デザインのポイント】
- ボーダー: border-b, border-gray-200
- 影: shadow-sm（テーブル全体）
- ステータスバッジ: 色分け（成功/警告/エラー）
- アイコンボタン: サイズ統一（w-8 h-8）

【ステータスの色分け】
- 承認済み: green
- 保留中: yellow
- 却下: red
- 下書き: gray

【技術スタック】
Next.js 14, TypeScript, Tailwind CSS, shadcn/ui, TanStack Table

【参考イメージ】
- GitHub Issues リスト
- Stripe Payments テーブル
- Linear Issues view
```
